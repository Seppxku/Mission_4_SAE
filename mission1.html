Mission 1: Conception – Interrogation d’une base de données

Pour cette première mission, l’objectif était de se familiariser avec les outils et données que l’on possède.

Avant le début de toute manipulation de données, nous devions en premier lieu découvrir la base de données de Kdou ainsi que le Query By Example (QBE) dans Microsoft Access qui nous permet de manipuler les données présentes dans la base.

Pour se faire, nous avons constitué différentes requêtes plus ou moins complexes permettant de faire le lien entre les différentes tables constituant la base de données de Kdou.

Après avoir localisé les clés primaires et les potentielles clés étrangères de chaque table, nous avons d’abord effectué des requêtes nécessitant une seule table :

  1. Obtenir le nombre de client ou de fournisseur se trouvant dans différentes régions du monde ou de la France (clients nantais, fournisseur nord-américains...).
  2. Obtenir des adresse de fournisseurs.
  3. Obtenir les différentes fonctions des fournisseurs, voir image ci-dessous :

Cependant, lors de la manipulation des données, nous nous sommes rendus comptes qu’il y avait certaines incohérences dans la base de données, ce qui pourrait fausser nos résultats.

Nous pouvons citer la présence de doublons tels que Sao Paulo et São Paulo qui étaient considérées comme deux villes différentes. Il y avait aussi et surtout de nombreuses fautes d’orthographe, rendant la recherche par nom de client très laborieuse.

Après cela, nous avons effectué des requêtes nécessitant les données présentes dans différentes tables. L’objectif étant de s’approprier la notion et le fonctionnement de la jointure :

  • Obtenir les informations de produits commandés par un certain client.
  • Obtenir l’offre de produits suédois.
  • Obtenir les catégories de produits vendus par des fournisseurs de différentes nationalités.

Afin d’éviter ce genre de soucis, nous avons proposé ces améliorations :

  1. Imposer une unique syntaxe pour les pays qui pourraient être écrits tout en majuscule et sans accent par exemple.
  2. Imposer une limite minimale de 0 pour exprimer l’absence d’un produit dans le stock dans l’onglet produit car il y a des enregistrements sans valeur portant ainsi confusion.
  3. Vérifier les codes clients dans la feuille des détails de commandes pour empêcher les soucis de gestion de commandes.

En conclusion, cette première mission avait pour objectif principal de nous faire découvrir les bases de données et la manipulation de celles-ci. De plus, les différentes requêtes saisies nous ont permis de nous rendre compte que l’utilisation de QBE est assez limitée.

Cet outil est utile pour la prise en main mais sera vite obsolète lorsque nous souhaiterons saisir des requêtes beaucoup plus complexes dans les missions suivantes. D’où l’intérêt d'utiliser le langage SQL.
